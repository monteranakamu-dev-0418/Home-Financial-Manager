# 要件定義書 — 家計管理アプリ

**バージョン：** 1.0
**最終更新：** 2026-04-19

---

## 1. 背景・目的

二人暮らしの家計を現金で管理しているが、出納の記録・確認が手間。
スマホから手軽に入力・確認でき、支出の可視化と立替精算ができるシステムを構築する。

---

## 2. ユーザー

| ユーザー | 役割 |
|--------|------|
| 中村 | 入力・閲覧 |
| 寺本 | 入力・閲覧 |

### ログイン仕様

- アプリ起動時に「中村 / 寺本」を選択するだけ
- パスワード不要（家庭内利用のため簡易設計）
- 選択後は localStorage に保持（次回起動時も維持）

---

## 3. 資金フロー

- 毎月二人でカテゴリ別に予算を設定して管理

---

## 4. 機能要件

### 4-1. 支出入力

- 入力項目：**カテゴリ・金額・支払方法・場所（任意）・日付・メモ（任意）**
- 支払方法：**現金 / PayPay / カード**
- 場所：過去に入力したことのある場所を予測候補として表示し、選択可能
- 日付：デフォルト「今日」
- 入力者：ログイン中のユーザーを自動セット（変更可能）

### 4-2. 立替管理

- PayPay・カードで支払った場合、個人立替として記録できる
- 立替記録の内容：立替者・金額・立替日・内容・支払方法・場所（任意）
- ステータス：**未精算 / 精算済**
- 精算完了時はステータスを「精算済」に変更するだけ（新規レコード作成なし）
- 精算前に確認ダイアログを表示（誤タップ防止）
- 立替タブ内で「未精算」「精算済」をタブ切替で表示

### 4-3. 月次予算管理

- カテゴリごとに月次予算を設定できる
- 前月の予算を一括コピーする機能あり
- 集計・ホーム画面で「予算 vs 実績」を進捗バーで表示
- 予算超過カテゴリを赤表示で強調

### 4-4. 支出一覧

- 通常支出＋精算済立替を表示（未精算立替は除外）
- 精算済立替には「立替」バッジを表示
- 月・カテゴリでフィルタ、キーワード（場所・メモ）検索
- 日付・金額・カテゴリ・支払方法・支払者でソート可能
- 各レコードから修正・削除が可能

### 4-5. 集計・残高確認

- 月ごとの残高（予算 - 支出合計）
- 残高がマイナスの場合は赤カード表示
- 先月以前の資産合計（累積キャリーオーバー）
- カテゴリ別実績 vs 予算
- 未精算立替サマリー（件数・合計金額）

### 4-6. グラフ可視化

- カテゴリ別支出の月次推移（折れ線グラフ）
- 全体支出の月次推移（折れ線グラフ）
- 月内カテゴリ内訳（ドーナツグラフ）
- 年内カテゴリ内訳（ドーナツグラフ）

### 4-7. カテゴリ管理

- カテゴリの追加・削除が設定画面から可能

---

## 5. 非機能要件

| 項目 | 要件 |
|------|------|
| デバイス | iPhone（Safari）からフル操作可能 |
| 常時稼働 | 自宅PCの常時起動不要（Vercel ホスティング） |
| コスト | 月額 ¥0（Vercel 無料枠・Supabase 無料枠） |
| 認証 | ユーザー選択のみ（パスワードなし） |
| PWA | ホーム画面追加でアプリ風動作（iOS 16.4+ で最適） |
| データ保持 | Supabase 無料枠の範囲内（500MB） |

---

## 6. アーキテクチャ

```
[iPhone Safari / PWA]
        ↕
[Next.js App (Vercel)]
        ↕
[Supabase PostgreSQL]
```

| 要素 | 採用 | 理由 |
|------|------|------|
| フロントエンド | Next.js + Vercel | 無料、PCなしで公開、PWA対応 |
| DB | Supabase | 無料枠500MB、PostgreSQL |
| 認証 | なし（ユーザー選択のみ） | シンプル設計 |

| 立替編集 | `/advances/[id]` | 立替の修正・削除 |

# 要件定義書 — 家計管理アプリ

**バージョン：** 0.2
**作成日：** 2026-04-18

---

## 1. 背景・目的

二人暮らしの家計を現金で管理しているが、出納の記録・確認が手間。
スマホから手軽に入力・確認でき、支出の可視化と立替精算ができるシステムを構築する。

---

## 2. ユーザー

| ユーザー | 役割 |
|--------|------|
| 中村 | 入力・閲覧 |
| 寺本 | 入力・閲覧 |

### ログイン仕様

- アプリ起動時に「中村 / 寺本」を選択するだけ
- パスワード不要（家庭内利用のため簡易設計）
- 選択後はセッションに保持（毎回選択不要）

---

## 3. 資金フロー

- 毎月25日に二人それぞれ **15万円を拠出**（合計30万円）
- カテゴリは今後追加の可能性あり

| カテゴリ |
|--------|
| 家賃 |
| 光熱費 |
| 水道代 |
| 食費 |
| 交際費 |
| 貯金 |

---

## 4. 機能要件

### 4-1. 支出入力

- 入力項目：**カテゴリ・金額・支払方法・場所・日付**
- 支払方法：**現金 / PayPay / カード**
- 場所：過去に入力したことのある場所を予測候補として表示し、選択可能
- 日付：デフォルト「今日」
- 入力者：ログイン中のユーザーを自動セット
- 入力ステップは **3タップ以内** を目標

### 4-2. 立替管理

- PayPay・カードで支払った場合、**個人立替**として記録できる
- 立替記録の内容：立替者・金額・立替日・内容・精算ステータス（未精算／精算済）
- 集計画面に「現在の立替残高」を表示（例：「中村が寺本に2,500円を立替中」）
- 精算完了時にステータスを「精算済」に変更できる

### 4-3. 月次予算管理

- 毎月25日前後に、カテゴリごとの**目標予算を設定**する
- 集計画面で「予算 vs 実績」を表示
- 予算超過カテゴリを視覚的に強調（赤表示など）

### 4-4. 支出一覧

- 日付・カテゴリ・金額・支払方法・場所・入力者を一覧表示
- カテゴリ・月・支払方法でフィルタリング可能

### 4-5. 集計・残高確認

- 月ごとのカテゴリ別合計
- 月の残高（30万 - 支出合計）
- 貯金残高の累計
- 立替残高サマリー
- **月ごとの平均支出（全体）**
- **月ごとの平均支出（カテゴリ別）**

> 平均は「アプリ利用開始月〜選択月」の期間で算出

### 4-6. グラフ可視化

- 月別支出の推移（全体）
- カテゴリ別支出の推移
- 月内のカテゴリ内訳（円グラフ）

### 4-7. 月次拠出記録

- 毎月25日の拠出（各15万）を記録

---

## 5. 非機能要件

| 項目 | 要件 |
|------|------|
| デバイス | iPhone（Safari）からフル操作可能 |
| 常時稼働 | 自宅PCの常時起動不要 |
| レスポンス | 入力・閲覧は3秒以内 |
| データ保持 | 最低2年分 |
| 認証 | ユーザー選択のみ（パスワードなし） |
| 通知 | 不要 |

---

## 6. アーキテクチャ方針

### 推奨構成

```
[iPhone Safari]
      ↕
[Webアプリ (Next.js)]
  ホスト：Vercel（無料）
      ↕
[DB：Supabase（無料）PostgreSQL]
```

| 要素 | 採用案 | 理由 |
|------|--------|------|
| フロントエンド | Next.js + Vercel | 無料、PCなしで公開、PWA対応 |
| DB | Supabase | 無料枠500MB、PostgreSQL |
| 認証 | なし（ユーザー選択のみ） | シンプル設計 |
| 開発環境 | Windows PC（ローカル） | 開発時のみ起動 |

> **月額コスト見込み：¥0**
